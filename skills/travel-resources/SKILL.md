# 出行资源查询技能

本 skill 是支撑 skill，负责为旅行、差旅、通勤、探店和普通外出补齐出行事实，供上层规划能力使用。所有 MCP 工具调用必须传 `user_id`。

## 边界

- 只查询和整理出行资源，不创建机票/酒店订单、不支付、不退改签、不收集乘机人或住客证件信息。
- 用户说“帮我订”“直接下单”时，说明当前可以继续比选或整理预订清单，但不能替用户下单。
- 不展示内部 token、供应商原始 JSON、Authorization、订单令牌或可直接下单的字段。
- 价格、库存、退改、早餐和房型以查询结果为准；不编造实时可售状态。

## 出行相关工具目录

| 工具 | 用途 |

| `location_geocode_address` | 地址、店名、景点名转坐标候选 |

| `travel_hotel_search` | 查询酒店列表 |
| `travel_hotel_rooms` | 查询酒店房型详情 |

## 工具缺失时

- 如果上表中的工具在当前 Agent 环境里不可用，优先使用可用的扩展工具发现/获取能力寻找等价工具或连接器，再继续完成查询。
- 扩展工具必须满足当前任务的安全边界：只读查询优先，不引入订单创建、支付、证件采集或会改变用户账户状态的能力。
- 找不到等价工具时，不要编造事实；改用已知上下文、保守假设和需要用户补充的信息继续规划。
- 面向用户回复时不解释内部工具缺失、工具名或获取过程，只说明哪些事实暂时无法确认以及下一步可以怎么补齐。

## 使用方式

- 地点/路线/天气：交给地图与位置工具；路线规划必须先把自然语言地点转成坐标。
- 机票：先确认出发地、目的地、日期、单程/往返、人数；城市名不确定时先查机场代码，再查航班。
- 酒店：先确认目的地、入住/离店日期、人数/房间数、预算、区域、星级、设施和出行场景；缺少关键日期时询问。
- 实时开放信息：营业时间、门票、展览、交通公告、入境/签证/政策等，用外部搜索或可用扩展工具确认。
- 查询结果用于规划时，优先概括推荐项，不堆砌全部候选。
- 相似度搜索不要无限翻页：机场/城市、POI、酒店、网页搜索这类按相关性排序的查询，最多看前两页或前两轮结果；如果仍找不到合适候选，应说明该结果大概率不存在、名称可能不准或供应商暂无库存，并改为请用户确认关键词/区域/日期。
- 不要为了“证明没有”反复换近似词或连续翻页；只有用户给了新的关键词、区域、日期或明确要求扩大范围时，才重新搜索。

## 输出重点

- 机票：航司/航班号、起降机场与时间、耗时、经停、最低价、座位状态、行李/退改摘要、是否需要用户到供应商侧确认最终价格。
- 酒店：名称、区域/地址、价格、评分、星级、距离、早餐/停车/Wi-Fi、适合场景、房型可用性。
- 地点与路线：地点名称、地址、距离、耗时、换乘或驾驶风险、天气影响。
- 结尾给下一步：继续筛选、整理预订清单、生成行程清单、保存计划或安排到日历。
